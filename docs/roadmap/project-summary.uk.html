Стан проєкту

Публічний handover snapshot із навмисно зафіксованою межею Rust-міграції.

Репозиторій є вимірюваним handover для Rust-міграції QSOE OS, а не переписуванням системи заради самого Rust. Rust тепер відповідає за низькоризикову leaf-логіку, парсери, host tools, userland services та task-manager providers, де це реально зменшує конкретні C-ризики. Kernel, spawn, capability, loader та seL4 authority шляхи навмисно залишені в C і мають статус deferred.

Канонічні адреси

Завершена міграція

  • Host qrvfs tools стали Rust-only: `qrvfs-tree` та `mkfs-qrv-rs`.
  • Userland services `slogger`, `pipe` та `devb-virtio` стали Rust-only.
  • `test_msgpass` став Rust-only у test images.
  • Retired task-manager providers лінкуються через спільний архів `qsoe-tm-providers`.

Стан task-manager providers

Retired C provider набір включає `tm_procfs`, `tm_cpio`, `tm_cred`, `tm_script`, `tm_elf`, `tm_reloc`, `tm_fdt`, `tm_syscfg`, `tm_sysmap`, `tm_sysfs`, `tm_pathmgr`, `tm_pseudodev` та `tm_rsrcdb`.

`tm-log` завершено в межах Rust-default formatter scope. Rust відповідає за typed formatter; C навмисно зберігає variadic ABI, `va_list` marshalling, LQ debug-console sink та explicit fallback selector.

Свідомо відкладено

  • Kernel Rust залишається deferred.
  • Spawn, capability, relocation та loader ownership залишаються C-owned.
  • High-risk runtime і kernel-adjacent робота потребує окремого milestone та risk review.
  • Подальше C-to-Rust перенесення не планується лише заради збільшення Rust coverage.
