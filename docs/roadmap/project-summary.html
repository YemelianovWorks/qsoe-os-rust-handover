Project State

Public handover snapshot with a deliberate Rust stop boundary.

TL;DR: Rust migration is complete for the practical low-risk scope: tools, services, parsers, and task-manager providers. Kernel, spawn, capability, loader, and seL4 authority boundaries remain in C by design.

The repository is a measurable Rust migration handover for QSOE OS, not a rewrite. Rust now owns the low-risk leaf logic, parsers, host tools, userland services, and task-manager providers where it reduces concrete C failure modes. Kernel, spawn, capability, loader, and seL4 authority paths remain C-owned and deferred by design.

  • Roadmap: GitHub Pages dashboard
  • Roadmap issues: issue-backed tracker
  • NQ mirror branch: dmytro-yemelianov/nq PR #1

Migration completed

  • Host qrvfs tools are Rust-only: `qrvfs-tree` and `mkfs-qrv-rs`.
  • Userland services `slogger`, `pipe`, and `devb-virtio` are Rust-only.
  • `test_msgpass` is Rust-only in test images.
  • Retired task-manager providers link through the shared `qsoe-tm-providers` archive.

Task-manager provider posture

The retired C provider set includes `tm_procfs`, `tm_cpio`, `tm_cred`, `tm_script`, `tm_elf`, `tm_reloc`, `tm_fdt`, `tm_syscfg`, `tm_sysmap`, `tm_sysfs`, `tm_pathmgr`, `tm_pseudodev`, and `tm_rsrcdb`.

`tm-log` is complete for its Rust-default formatter scope. Rust owns the typed formatter; C intentionally keeps the variadic ABI, `va_list` marshalling, LQ debug-console sink, and explicit fallback selector.

Deferred by design

  • Kernel Rust remains deferred.
  • Spawn, capability, relocation, and loader ownership remains C-owned.
  • High-risk runtime and kernel-adjacent work needs a separate milestone and risk review.
  • No further C-to-Rust translation is planned merely to increase Rust coverage.
